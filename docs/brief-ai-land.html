Автоматизація — флоу

PIPELINE 01 Weekly Research → Telegram → Publish
⏰ Щотижневий тригер → 🔍 Пошук нових AI тулів → 🤖 Claude API: опис, ELI5, теги, юз кейси → 📸 Screenshot прев'ю → 📲 Telegram повідомлення → 👍 Реакція команди → ✅ Авто-публікація на сайті
Оркестрація: Make.com · Зберігання: Supabase або Airtable · Скрін: ScreenshotOne API або Playwright
PIPELINE 02 User Submission → Telegram → Publish
📝 Submit Tool форма → 🤖 Auto-enrich (Claude API) → 📲 Telegram [USER SUBMITTED] → 👍 Реакція = approved → ✅ Публікація на сайті
Той самий Telegram канал, інша позначка. Помилкові submission фільтруються до відправки в Telegram.
FEATURE 01 "How can this help me?" — Discuss with AI
💬 Кнопка на detail сторінці → Генерується промпт з контекстом тулу → ChatGPT / Claude / Perplexity
Референс: thegradient.com — кнопки внизу статті "Discuss with your AI". Логіка на фронтенді, без API.
FEATURE 02 Search — SQL фільтрація
⌘K пошук → Full-text search по назві, опису, юз кейсам → Фільтр по категорії / тегах → Результати в реальному часі
Supabase full-text search або Algolia (залежно від БД). Не LLM.

Рекомендований стек

Сайт
Next.js + Vercel
Прототип вже на Next.js. Потрібні динамічні маршрути, SSR, API routes — Webflow тут надто обмежений.
База даних / CMS
Supabase
PostgreSQL full-text search, REST API для Make.com, реальний час. Airtable — як альтернатива якщо простіше для команди.
Автоматизація
Make.com
Visual workflow, Telegram модуль, Supabase connector, Claude API. n8n — як self-hosted альтернатива.
AI Research
Claude API + Perplexity API
Claude для генерації описів/ELI5/тегів. Perplexity для пошуку нових тулів по актуальним джерелам.
Скріншоти
ScreenshotOne API
Простий API для screenshot по URL. $19/mo для weekly обсягів. Зберігаємо в Supabase Storage.
Telegram Bot
Telegram Bot API + Make.com
Make.com слухає реакції через Telegram trigger. Reaction → webhook → Supabase update → publish.

Уточнюючі питання

Критичні — потрібні перед естімейтом
1 Дизайн-файл (Figma) буде готовий до початку розробки — чи розробляємо паралельно з дизайном? CRITICAL
2 Які джерела для weekly research? Product Hunt, конкретні сайти, соцмережі — чи визначаємо разом? CRITICAL
3 Прев'ю тулу — автоматичний скріншот сайту, або лого/ручне зображення? (впливає на pipeline складність) CRITICAL
Технічні
4 Є налаштований Telegram бот/канал, чи створюємо з нуля?
5 Бюджет на API сервіси (Make.com ~$16/mo, ScreenshotOne ~$19/mo, Claude API ~$20-50/mo)?
6 Є Supabase / Vercel акаунти, чи налаштовуємо у вас?
Скоуп / Контент
7 ELI5 опис та теги — генерує AI автоматично, чи команда редагує перед публікацією?
8 Стартовий контент — 150+ тулів потрібно залити вручну, чи ми мігруємо з прототипу?
9 Чи потрібна можливість редагувати тул після публікації (без Telegram)?
Дедлайни
